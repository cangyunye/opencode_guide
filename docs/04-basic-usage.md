# 基础使用场景

难度：基础

现在 OpenCode 已经在你的项目里跑起来了。这一章介绍几个最常见的使用场景，帮你快速上手。

## 场景一：让 AI 解释代码

接手一个陌生项目，看到一堆代码不知道什么意思？直接问 OpenCode：

```
@src/auth/login.ts 这个文件做了什么
```

`@` 符号用来引用项目中的文件。OpenCode 会读取这个文件的内容，然后用你能听懂的话解释它。

你也可以问更具体的问题：

```
@src/auth/login.ts 这里的 token 验证逻辑是怎么工作的？为什么要在第 42 行检查过期时间？
```

OpenCode 会针对你问的部分做详细解释。

## 场景二：直接修改代码

如果改动很简单，不需要多想，直接告诉 OpenCode：

```
@src/utils/format.ts 把 formatDate 函数改成支持 yyyy-MM-dd 和 MM/dd/yyyy 两种格式
```

OpenCode 会读取文件、理解现有代码、做修改。改完后你会在界面里看到它改了哪些地方（类似 Git diff）。

如果改得不对，用 `/undo` 撤销，调整你的描述再试一次。

## 场景三：添加新功能

稍微复杂一点的需求，可以直接说：

```
帮用户列表页加一个搜索框，支持按姓名和邮箱搜索
```

OpenCode 会自己去找用户列表相关的文件，理解现有结构，然后实现搜索功能。它可能会：
1. 先读取相关文件
2. 在组件中添加搜索输入框
3. 添加搜索过滤逻辑
4. 告诉你它做了什么改动

对于更复杂的功能，建议先用计划模式规划方案，再让它实施。这在进阶篇的[计划模式与构建模式](08-plan-vs-build.md)中会详细讲。

## 场景四：撤销不满意的改动

AI 改错了，或者改得不是你想要的，不要慌：

```
/undo
```

这个命令会撤销 AI 最近一次操作引起的所有文件变更。你可以多次使用 `/undo` 一直撤回到你满意的状态。

如果撤多了，用 `/redo` 恢复。

## 场景五：调试报错

程序报了一个错，你搞不定？直接把错误信息粘贴给 OpenCode：

```
运行 npm run build 报了这个错：
TypeError: Cannot read properties of undefined (reading 'map')
  at UserList (src/components/UserList.tsx:25:18)
帮我看看怎么修
```

OpenCode 会读取 `UserList.tsx` 第 25 行附近的代码，分析错误原因，给出修复方案。

## 场景六：让 AI 执行终端命令

你可以在对话中用 `!` 直接执行终端命令，并把结果纳入对话上下文：

```
!npm test
```

OpenCode 会运行命令，把输出展示给你看。它还会根据输出判断有没有测试失败，并主动提出修复建议。

这在调试时特别有用 -- 你不需要切换到另一个终端窗口去跑命令。

## 快捷键速查

这些快捷键在日常使用中会频繁用到：

| 快捷键 | 功能 |
|--------|------|
| `Tab` | 切换计划模式 / 构建模式 |
| `Ctrl+X N` | 新建会话 |
| `Ctrl+X L` | 查看和切换会话 |
| `Ctrl+X U` | 撤销上次操作 |
| `Ctrl+X R` | 重做撤销的操作 |
| `Ctrl+X C` | 压缩当前上下文 |
| `Ctrl+X M` | 查看可用模型 |
| `Ctrl+X S` | 分享当前会话 |
| `Ctrl+X I` | 运行 /init |
| `Ctrl+X E` | 用外部编辑器写消息 |
| `Ctrl+X H` | 查看帮助 |
| `Ctrl+X Q` | 退出 OpenCode |

不需要死记硬背。用到的时候按 `Ctrl+X H` 查看帮助就行。
