# 技能系统入门

难度：进阶

## 为什么需要技能

没有技能的 AI 是一个"什么都能聊一点但什么都不精"的通用助手。它可能帮你写代码，但不会遵循特定的开发流程。写出来的代码可能能用，但测试覆盖不够、命名不一致、不符合你的项目规范。

**技能（Skills）** 就是给 AI 装备的"工具箱"。安装了 TDD 技能后，AI 会严格按照"先写测试、再写代码、再重构"的流程工作。安装了 code-review 技能后，AI 会按特定的检查清单审查代码质量。

通过不同的技能组合，你可以把 AI 塑造成符合你团队工作方式的编程搭档。

## 技能是什么

每个技能就是一个 `SKILL.md` 文件，里面写着一套行为规则。比如"遇到这种任务时，按这些步骤来""输出内容必须是这种格式""必须遵守这些原则"。

技能放在特定目录下，OpenCode 启动时会自动发现它们：

- 项目级：`.opencode/skills/<技能名>/SKILL.md`
- 全局级：`~/.config/opencode/skills/<技能名>/SKILL.md`

## 技能从哪来

三个来源：

**内置技能**：OpenCode 自带了一些技能，开箱即用。你不需要安装，它们已经在那里了。

**社区技能**：其他开发者或团队发布的技能，可以通过命令安装：

```bash
npx skills add <作者>/<仓库> --skill=<技能名>
```

比如安装 grill-with-docs 技能：

```bash
npx skills add mattpocock/skills --skill=grill-with-docs
```

**自定义技能**：你自己编写的技能，放在 `.opencode/skills/` 目录下。针对你项目的特殊需求量身定制。

## 技能的触发机制

技能不需要你手动"启动"。AI 会根据你的请求内容，自动判断是否需要加载某个技能。

### 自动触发

AI 在处理你的请求时，会查看可用技能的描述（description）。如果你的请求和某个技能的描述匹配，AI 会自动加载那个技能的完整内容，然后按照技能规定的流程来工作。

比如你安装了 TDD 技能，然后说"帮我给用户服务添加一个批量删除方法"，AI 检测到这是一个编码任务，与 TDD 技能的描述匹配，就会自动加载 TDD 技能，按照"先写失败测试 -> 再写代码通过测试 -> 重构"的流程来操作。

你不需要说"请使用 TDD 技能"，AI 自己会判断。

### 强制触发

有时候 AI 没有自动加载你想要的技能（可能是因为请求描述不够明确），你可以明确要求：

```
请使用 writing-plans 技能，帮我制定一个添加支付功能的实施计划
```

### 确认技能是否已加载

怎么知道 AI 有没有用上某个技能？两个方法：

1. **观察 AI 的行为特征**：每个技能都有特定的行为模式。比如 brainstorming 技能会"一次只问一个问题"；TDD 技能会"先写测试文件"。如果 AI 的行为符合某个技能的特征，说明它已经加载了。
2. **用 /details 查看工具调用**：按 `Ctrl+X D` 切换工具执行详情显示，你会看到 AI 调用了哪些工具。如果列表中有 `skill` 工具的调用记录，说明 AI 加载了某个技能。

### 触发失败的常见原因

- 技能被设置为 `deny`（在权限配置中）
- 技能的 description 写得太模糊，AI 无法匹配到你的请求
- 技能文件路径错误，OpenCode 没有发现它
- 技能 description 和实际功能不符，AI 误判不需要

## 技能权限控制

你可以控制每个技能是否可用：

- `allow`：AI 随时可以加载这个技能
- `ask`：AI 加载前会提示你确认
- `deny`：AI 不能使用这个技能

在权限配置中设置：

```json
{
  "permission": {
    "skill": {
      "internal-*": "allow",
      "experimental-*": "deny",
      "*": "ask"
    }
  }
}
```

## 深度分析：为什么技能决定了 AI 能走多远

> 同样的 DeepSeek 模型，为什么有人做任务很快就中断，有人却能持续很久？
>
> 完整分析见 **[为什么技能决定了 AI 能走多远](09a-why-skills-matter.md)**

**摘要**：中断的核心原因分三层。第一层是**模型层**，DeepSeek 等低价模型的上下文保持能力偏弱、自主判断"完成"的阈值低、多步推理和工具调用链较短。第二层是**行为层**，没有技能约束时 AI 的默认模式是"做一步等一步"，每步都停下来等用户反馈。第三层是**结构层**，所有计划、决策都存在对话上下文中，一旦上下文压缩就全部丢失。技能通过三个机制逐层解决：writing-plans 把计划持久化到磁盘文件（解决记忆丢失）、brainstorming 前置锁定需求消除返工（解决上下文浪费）、TDD 内置自纠错让 AI 不依赖人工就能自我修复（解决步骤中断）。三者组合形成自适应工作流，让 AI 从"被动等待"变成"有计划地持续推进"。
