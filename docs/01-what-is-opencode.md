# OpenCode 是什么

难度：基础

## 一句话介绍

OpenCode 是一个开源的 AI 编码助手。它住在你的终端里，能读你的代码、改你的文件、跑你的命令 -- 就像一个随时待命的编程搭档。

## 它和 ChatGPT 有什么区别

你可能用过 ChatGPT 写代码：你问它一个问题，它给你一段代码，你自己复制粘贴到编辑器里。这个过程中，ChatGPT 只是个"回答问题的工具"，所有动手操作都是你来做。

OpenCode 不一样。它是 **Agent**（智能体），不只是回答问题，还会自己执行操作：

- 它能读取你项目里的文件，理解你的代码结构
- 它能直接修改文件，把代码写进你的项目
- 它能运行终端命令，帮你测试和构建
- 它能把一系列改动整合成完整的方案，而不是零散的代码片段

打个比方：ChatGPT 像是一本编程 cookbook，你照着菜谱自己做菜；OpenCode 像是一个厨师助手，你告诉它想吃什么，它帮你把菜做出来。

## 核心特性

**模型中立**：OpenCode 不绑定某一家 AI 公司。你可以用 Claude、GPT、Gemini，也可以用 DeepSeek、MiniMax 等国产模型，还可以用本地模型。切换模型只需要一行配置。

**多会话**：同一个项目可以同时开多个对话窗口，互不干扰。一个会话里在做前端页面，另一个会话里在写后端接口，并行推进。

**技能系统**：通过安装不同的"技能"，你可以让 AI 掌握特定的工作方法。比如安装 TDD 技能后，AI 会严格按照"先写测试、再写代码"的流程工作。

**权限控制**：你可以决定 AI 能做什么、不能做什么。比如允许它读文件但不允许删除文件，允许运行 git 命令但不允许运行 rm 命令。

**开源**：代码完全开放，任何人都可以查看、使用和贡献。目前在 GitHub 上有超过 70,000 颗星。

## 三种使用方式

OpenCode 提供三种形态，功能完全一致，选你喜欢的：

**终端界面（TUI）**：在终端里运行的交互界面。这是最经典的使用方式，适合习惯命令行工作的开发者。

**桌面应用**：独立的桌面程序，支持 macOS、Windows 和 Linux。有图形化的设置面板和会话管理，适合更喜欢窗口界面的用户。

**IDE 扩展**：集成到 VS Code 等编辑器中。不需要切换窗口，直接在编码环境中使用。

这三种方式共享同一套配置文件和项目设置。你在终端里配置好的供应商和规则，桌面版和 IDE 版都能直接用。
