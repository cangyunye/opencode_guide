# 计划模式与构建模式

难度：进阶

OpenCode 有两种主工作模式：**Plan**（计划）和 **Build**（构建）。用 `Tab` 键一键切换。你会在右下角看到当前模式标识。

## Plan 模式：只看不改

在 Plan 模式下，AI 能读取文件、搜索代码、分析结构，但**不会修改任何文件或执行有副作用的命令**。它的任务是帮你理解代码和制定方案。

适合的场景：
- 接手陌生代码，想先搞清楚架构再动手
- 要做一个复杂功能，需要先规划实现路径
- 不确定怎么改，想让 AI 先分析几种方案

用法：

```
<Tab>          # 切换到 Plan 模式
```

然后描述你的需求：

```
我想给这个项目加一个用户权限系统，支持角色管理和功能权限控制。请先分析一下现有的代码结构，然后给出实现方案。
```

AI 会读取你的项目文件，分析结构，然后给出一份详细的方案。你可以针对方案提问、要求修改、补充细节。

## Build 模式：直接动手

Build 是默认模式。AI 拥有完整的工具权限：读文件、写文件、运行命令，全部可以。

适合的场景：
- 需求明确，直接让 AI 实施
- Plan 模式讨论完方案，切换过来执行

## 推荐工作流：先 Plan，再 Build

这是使用 OpenCode 最推荐的工作方式，尤其是对于不太简单的任务：

```
<Tab>                              # 切换到 Plan

我需要在用户列表页添加分页功能，每页显示 20 条，支持跳转到指定页码。
请先看一下现有的列表组件和数据获取逻辑，然后给出实现方案。

（AI 分析代码，给出方案）
（你提出修改意见，AI 调整方案）
（方案满意后）

<Tab>                              # 切换回 Build

方案没问题，开始实施吧。
```

这种方式的好处是避免 AI 一上来就改错地方。先对齐思路，再动手，效率更高。

## 简单任务直接 Build

不是所有任务都需要先 Plan。改个拼写错误、加一行日志、调个样式这种简单的改动，直接在 Build 模式下说就行：

```
@src/components/Header.tsx 把 logo 的大小从 32px 改成 48px
```

原则：改动能在 5 分钟内完成且不太可能出错，直接 Build。需要思考、涉及多文件、可能需要讨论的，先 Plan。

## 子代理

除了 Plan 和 Build 两种主模式，OpenCode 还有几个子代理，AI 可以在需要时自动调用。

**Explore**：快速只读搜索。用来查找文件、搜索关键词。当你问"项目中有没有处理用户认证的代码"时，AI 可能会启动一个 Explore 子代理去搜一遍。

**Scout**：外部文档研究。用来查阅依赖库的文档、了解第三方 API 的用法。当你问"Next.js 14 的 Server Actions 怎么用"时，AI 可能启动 Scout 去查。

**General**：通用多步骤任务。拥有完整的文件操作权限，可以独立执行一个子任务。当你给了一个较大的需求时，主代理可能把其中独立的步骤分派给 General 子代理并行处理。

你不需要手动调用子代理，AI 会根据需要自动使用。但在 Plan 模式下，Explore 和 Scout 会更常用，因为 Plan 模式本身就是只读的。
