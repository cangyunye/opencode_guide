# 快捷键速查

难度：基础

OpenCode 提供了完整的键盘快捷键系统，掌握它们可以大幅提升操作效率。本章整理了所有官方支持的快捷键，并按功能分类。

## 前导键机制

OpenCode 使用 **前导键（Leader Key）** 来避免与终端快捷键冲突。

默认前导键：`Ctrl+X`

**用法**：按下 `Ctrl+X` → 释放 → 再按第二个字母键

示例：`Ctrl+X` → 释放 → `N` = 新建会话

> ⚠️ 注意：不要同时按住三个键。正确操作是"按 Ctrl+X → 松开 → 按字母"。

## 一、基础操作

| 快捷键 | 功能 |
|--------|------|
| `Enter` | 发送消息 |
| `Shift+Enter` | 换行（不发送） |
| `Ctrl+Enter` | 换行（不发送） |
| `Alt+Enter` | 换行（不发送） |
| `Ctrl+J` | 换行（不发送） |
| `Escape` | 中断 AI 响应（按两次强制中断） |
| `Ctrl+P` | 打开命令面板 |

### Ctrl+C 的三种行为

`Ctrl+C` 是最常用的快捷键之一，但它在不同场景下有不同行为：

| 场景 | 行为 |
|------|------|
| 对话框打开时 | 关闭对话框 |
| 输入框有内容时 | **清空输入内容**（不会退出） |
| 输入框为空时 | **退出 OpenCode** |

这与程序员熟悉的终端行为一致：`Ctrl+C` 优先清空当前输入，输入为空时才退出。

### Ctrl+D 与 Ctrl+C 的区别

| 快捷键 | 行为 |
|--------|------|
| `Ctrl+C` | 先尝试清空输入，输入为空时退出 |
| `Ctrl+D` | 直接发送退出信号，立即退出 |

## 十、自定义快捷键

在 `~/.config/opencode/tui.json` 中配置：

```json
{
  "$schema": "https://opencode.ai/tui.json",
  "keybinds": {
    "leader": "ctrl+x",
    "app_exit": "ctrl+c,ctrl+d,<leader>q",
    "session_new": "<leader>n",
    "model_list": "<leader>m"
  }
}
```

### 禁用快捷键

设置为 `"none"` 即可禁用：

```json
{
  "keybinds": {
    "session_compact": "none"
  }
}
```

### 配置文件位置

| 系统 | 路径 |
|------|------|
| Linux/macOS | `~/.config/opencode/tui.json` |
| Windows | `%USERPROFILE%/.config/opencode/tui.json` |
| 项目级别 | `.opencode/tui.json` |

## 常见问题

### Ctrl+C 不是复制

在 OpenCode 中，`Ctrl+C` 的作用是清空输入或退出，而不是复制。要复制消息，请使用 `Ctrl+X` → `Y`。

### 向上箭头不显示历史

只有当输入框为空时，向上箭头才会显示历史消息。如果输入框有内容，请先按 `Ctrl+C` 清空，再按向上箭头。

### 快捷键不起作用

检查终端是否拦截了这些快捷键。有些终端（如 Windows Terminal）可能需要在设置中禁用某些快捷键，或者使用 Alt 键替代 Ctrl 键。
