# 桌面版操作指南

难度：进阶

## 桌面版 vs 终端版

OpenCode 提供三种使用形态：终端界面、桌面应用、IDE 扩展。这三者功能完全一致，区别在于交互方式。

**终端版**：纯键盘操作，适合习惯命令行的开发者。轻量、快速、可 SSH 远程使用。

**桌面版**：图形界面，适合更喜欢窗口操作的用户。有侧边栏、设置面板、拖拽等图形化操作。

**IDE 版**：集成在编辑器中，适合不想切换窗口的开发者。

这一章专门介绍桌面版的操作。

## 安装桌面版

桌面版支持 macOS、Windows 和 Linux。

**macOS**：从 [OpenCode 官网](https://opencode.ai) 下载 .dmg 安装包，拖到 Applications 即可。

**Windows**：下载 .exe 安装包，按向导安装。推荐使用 Windows 原生版而非 WSL 版（除非你需要 WSL 中的开发环境）。

**Linux**：下载 .AppImage 或 .deb 包。AppImage 可以直接运行：

```bash
chmod +x OpenCode-*.AppImage
./OpenCode-*.AppImage
```

## 界面介绍

桌面版的界面分为几个区域：

**侧边栏**（左侧）：显示项目列表和会话列表。你可以在这里快速切换不同项目和不同会话。

**主对话区**（中央）：和 AI 的对话区域，和终端版一样可以发送消息、查看 AI 回复和工具执行结果。

**工具执行详情**（底部或右侧）：显示 AI 执行了哪些工具（读文件、写文件、运行命令等），和终端版的 `/details` 功能一样。

## 桌面版专属功能

### 图形化会话管理

侧边栏可以直接新建、切换、Fork 会话。不需要输入命令，点击即可。

### 拖拽图片

你可以直接把图片拖拽到对话区域。AI 会识别图片内容并纳入上下文。适合给 AI 展示设计稿、UI 截图、错误截图等视觉参考。

### 设置面板

点击设置图标进入图形化配置界面，可以：
- 添加和切换 AI 供应商（相当于终端版的 /connect）
- 选择和切换模型（相当于 /models）
- 切换主题和字体
- 配置权限规则

终端版需要编辑 JSON 文件的配置，桌面版可以通过图形界面完成大部分操作。

### 多项目侧边栏

桌面版可以在侧边栏管理多个项目，快速切换。终端版需要 `cd` 到不同目录然后启动 OpenCode。

### 字体和主题

桌面版提供可视化的字体选择器和主题切换。终端版需要通过配置文件或命令设置。

## 配置共享

桌面版和终端版共用同一套配置：

- `opencode.json`：项目配置，桌面版和终端版都能读取
- `AGENTS.md`：项目规则，两版共享
- `~/.config/opencode/`：全局配置，两版共享
- `.opencode/skills/`：技能文件，两版共享

你在终端版配置好的供应商和规则，打开桌面版就能直接用。反过来也一样。

## 什么时候用桌面版

**适合桌面版的场景**：
- 不太熟悉终端命令，更喜欢鼠标操作
- 需要频繁拖拽图片给 AI 看
- 需要同时管理多个项目
- 想通过图形界面配置设置

**适合终端版的场景**：
- 已经习惯命令行工作流
- 通过 SSH 远程连接服务器
- 需要在 tmux/screen 中和其他工具并排使用
- 追求启动速度和轻量化

两种方式可以混合使用。你在办公室用桌面版，远程连接时用终端版，数据和配置都是同步的。
