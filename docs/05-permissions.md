# 权限与安全基础

难度：基础

OpenCode 能直接读写你的文件、运行你的命令。这种能力很强大，但也需要一定的安全保护。OpenCode 内置了一套权限系统来控制风险。

好消息是：**默认设置已经足够安全，你不需要做任何额外配置就能开始使用。**

## 默认就安全的保护机制

日常使用中，以下保护机制在默认状态下就已经生效：

**.env 文件保护**：包含密钥和敏感信息的 `.env` 文件，默认被禁止读取。AI 不会偷看你的数据库密码或 API 密钥。`.env.local`、`.env.production` 等变体同样被禁止。只有 `.env.example` 可以被读取。

**重复操作自动暂停**：如果 AI 对同一个工具连续调用 3 次都做同样的事（比如反复修改同一个文件同一个位置），会自动暂停。这说明 AI 陷入了死循环，用 `/undo` 撤销后换一种描述方式即可。

**项目外路径提示**：当 AI 要访问你项目目录之外的文件时，会先征求你的同意。

**--auto 模式**：如果你信任 AI 的操作，启动时加 `opencode --auto`，可以自动批准权限请求。但即使开启 --auto，被明确禁止的操作（如读取 .env）仍然会被拒绝。

## 看到权限提示时怎么判断

绝大多数操作不会弹权限提示，AI 会直接执行。你可能遇到的提示只有这几种：

- **访问项目外的文件**：想想 AI 是否真的需要读那个路径
- **重复操作**：AI 陷入死循环了，拒绝并 /undo
- **危险命令**：如果将来你配置了额外规则（如禁止 rm 命令），违反时会提示

## 权限精细配置

当你需要更精细地控制 AI 的行为（比如允许 git 命令但禁止 rm 命令），可以配置 `opencode.json` 的 `permission` 字段。这在进阶篇的[权限系统详解](05a-permissions-advanced.md)中有完整说明，包含所有操作的默认权限值、配置方法和 glob 模式匹配规则。

对于刚入门，默认设置足够了。先正常使用，等遇到需要调整的场景再回来查阅。
