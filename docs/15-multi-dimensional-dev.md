# 多维度开发

难度：进阶

## 什么是多维度开发

"多维度开发"指的是同时推进多个独立或半独立的工作。比如：

- 同时开发前端页面和后端接口
- 同一个项目里一个人做功能 A，另一个人做功能 B
- 在不同分支上并行尝试不同的实现方案

在 OpenCode 中，你可以通过多会话、子代理和 Git Worktree 来实现并行开发。

## 多 Agent 运行前的准备

在让多个 Agent 同时工作之前，需要做好以下准备：

### 1. 确保项目已初始化

运行过 `/init`，AGENTS.md 内容完整。所有 Agent 共享同一份 AGENTS.md，这是它们理解项目的共同基础。

### 2. 规划好文件边界

明确哪个 Agent 负责哪些文件或模块。如果两个 Agent 同时修改同一个文件，会产生冲突。比如：

- Agent A：负责 `src/components/` 和 `src/hooks/`（前端）
- Agent B：负责 `src/api/` 和 `src/services/`（后端）

### 3. 确认权限配置

多 Agent 并行时，手动批准权限会成为瓶颈。确保需要并行的操作的权限设为 allow：

```json
{
  "permission": {
    "edit": {
      "*": "allow"
    },
    "bash": {
      "npm *": "allow",
      "git *": "allow",
      "*": "ask"
    }
  }
}
```

### 4. Git Worktree（可选）

如果需要在不同的 Git 分支上同时工作，提前创建 Worktree：

```bash
git worktree add ../my-app-feature-a feature-a
git worktree add ../my-app-feature-b feature-b
```

然后在不同的 Worktree 目录中分别启动 OpenCode。

### 5. 确认 API 配额

多个 Agent 同时调用模型 API，消耗量会成倍增加。确认你的供应商配额足够。

## 多会话并行

最简单的并行方式：开多个 OpenCode 会话。

```
# 会话 1：前端开发
/new
帮我实现用户列表页的前端组件，包括搜索框和分页

# 会话 2：后端开发
/new
帮我实现用户列表的后端 API，支持搜索和分页

# 切换会话
/sessions
```

在 `/sessions` 列表中可以快速切换。每个会话有独立的上下文，互不干扰。

## 子代理并行

在一个会话内，主 Agent 可以把独立的子任务分派给 General 子代理并行处理：

```
请同时完成以下三件事：
1. 给用户模型添加手机号字段
2. 创建用户头像上传接口
3. 编写用户列表的分页查询
```

AI 会识别这三个任务互相独立，可能分派给三个子代理同时执行。比串行执行快很多。

## 分享会话

完成某个维度的开发后，用 `/share` 生成分享链接，发给团队成员参考：

```
/share
```

## 团队协作建议

- **统一 AGENTS.md**：所有团队成员使用同一份 AGENTS.md，确保 AI 对项目的理解一致
- **统一 MCP 配置**：如果用外部工具（如数据库、部署服务），通过共享 opencode.json 配置 MCP 服务器
- **避免同文件冲突**：明确分工，不要让两个 Agent 同时改同一个文件
- **代码审查**：一个 Agent 完成后，另一个会话用 code-review 技能审查

## 什么时候用多维度开发

适合的场景：
- 前后端需要同时开发
- 多个独立功能需要并行推进
- 团队成员各负责不同模块

不适合的场景：
- 任务之间有强依赖（B 必须等 A 完成）
- 只有一个简单的改动
- 项目很小，并行没有意义
