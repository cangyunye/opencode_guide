# 会话管理与常用命令

难度：基础

## 什么是会话

OpenCode 中你和 AI 的每一次完整对话就是一个"会话"。一个会话记录了你们聊了什么、AI 做了哪些操作、改了哪些文件。

当你打开 OpenCode 开始对话时，一个新会话就创建了。当你退出再打开，OpenCode 会恢复上次的会话。

## 常用命令一览

以下是日常使用中最常用的命令：

### 新建和切换

| 命令 | 快捷键 | 功能 |
|------|--------|------|
| `/new` | `Ctrl+X N` | 创建新会话（当前会话保留） |
| `/sessions` | `Ctrl+X L` | 查看所有会话列表，选择切换 |
| `/exit` | `Ctrl+X Q` | 退出 OpenCode |

`/new` 会保留当前会话，开启一个全新的对话。适合你要做一件完全不同的事情时使用。

`/sessions` 会列出你所有的历史会话（按时间排序），你可以用上下箭头选择，回车切换到那个会话继续对话。

### 撤销和重做

| 命令 | 快捷键 | 功能 |
|------|--------|------|
| `/undo` | `Ctrl+X U` | 撤销 AI 最近一次操作及文件变更 |
| `/redo` | `Ctrl+X R` | 重做被撤销的操作 |

`/undo` 是你的安全网。AI 改错了就用它撤回来。可以连续多次撤销。

### 上下文管理

| 命令 | 快捷键 | 功能 |
|------|--------|------|
| `/compact` | `Ctrl+X C` | 压缩当前会话的上下文 |

随着对话越来越长，AI 需要处理的上下文会越来越大。`/compact` 会把之前的对话总结成一份简短的摘要，释放空间。当你感觉 AI 开始"忘记"之前聊过什么时，可以手动执行一下。

### 供应商和模型

| 命令 | 快捷键 | 功能 |
|------|--------|------|
| `/connect` | -- | 添加或切换 AI 供应商 |
| `/models` | `Ctrl+X M` | 查看可用模型列表 |

`/connect` 在你第一次配置时用过。之后如果想添加更多供应商（比如同时配置 Claude 和 GPT），可以随时再用。

`/models` 让你查看和切换当前可用的模型。怎么选模型是进阶话题，在[模型选择](07-model-selection.md)中详细讲。

### 分享和导出

| 命令 | 快捷键 | 功能 |
|------|--------|------|
| `/share` | `Ctrl+X S` | 生成当前会话的分享链接 |
| `/export` | `Ctrl+X X` | 导出对话为 Markdown 文件 |

`/share` 会生成一个链接，你可以发给同事，让他们在浏览器里看到你们的完整对话过程。注意：分享后任何人都能看到对话内容，不要分享含敏感信息的会话。

### 其他

| 命令 | 快捷键 | 功能 |
|------|--------|------|
| `/init` | `Ctrl+X I` | 扫描项目，生成或更新 AGENTS.md |
| `/help` | `Ctrl+X H` | 显示帮助信息 |

## 一个典型的使用流程

```bash
# 1. 进入项目目录，启动 OpenCode
cd ~/projects/my-app
opencode

# 2. 如果是新项目，先初始化
/init

# 3. 开始工作 -- 比如让 AI 修一个 bug
帮我修复 #142 号 issue 描述的登录超时问题

# 4. 如果改得不对，撤销
/undo

# 5. 调整描述，重新来
登录超时的问题，请先看一下 @src/services/auth.ts 的超时配置，然后...

# 6. 改好了，开个新会话做别的事
/new

# 7. 需要参考之前的会话
/sessions

# 8. 下班了，退出
/exit
```
